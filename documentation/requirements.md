## *<Λειτουργική Απαίτηση 1>*

 **Ο χρήστης πρέπει να μπορεί να δημιουργεί προσωπικό ηλεκτρονικό λογαριασμό στην υπηρεσία .**

# Περιγραφή: 
Εφόσον ο κύριος στόχος της υπηρεσίας είναι η επικοινωνία των ενδιαφερομένων για θέσεις εργασίας οι οποίες ειναι διαθέσιμες με τους φορείς απασχόλησης πρέπει να μπορούν να μπορούν να δημιουργούν λογαριασμό παρέχοντας βιογραφικό και στοιχεία επικοινωνίας.

# User Priority: (5/5) 
Η *<Λειτουργική Απαίτηση 1>* είναι βασική για τους διάφορους φορείς απασχόλησης, καθώς χωρίς προσωπικό λογαριασμό δεν γίνεται ο εργοδότης να επικοινωνεί με τον ενδιαφερόμενο για τη κάθε θέση.

# Technical Priority: (5/5)
 Η υλοποίηση της *<Λειτουργική Απαίτηση 1>* είναι αναγκαία και αποτελεί κύριο στόχο της υπηρεσίας.

# Stability: (5/5)
 Η δημιουργία λογαριασμού αν και μπορεί να πραγματοποιείται με  συγκεκριμένο τρόπο όπου δεν συνίσταται καμία μεταβολή ,μελλοντικά είναι ιδιαίτερα εύχρηστο να πραγματοποιηθεί και χωρίς την εγγραφή. Για παράδειγμα για την εγγραφή και την ταυτοποίηση τους θα μπορούσαν να χρησιμοποιούν τον προσωπικό τους λογαριασμό από κάποιο από τα μέσα κοινωνικής δικτύωσης .

[signup_account.feature](https://github.com/nstamatak/1o-Paradoteo/blob/master/requirements/signup_account.feature)


## *<Λειτουργική Απαίτηση 2>*

 **Το σύστημα πρέπει να επιτρέπει την είσοδο στο χρήστη με τη χρήση ονόματος λογαριασμού και συνθηματικού .**

# Περιγραφή: 
Σε περιπτώσεις παροχής τέτοιου τύπου υπηρεσιών πρέπει να υπάρχει ασφάλεια όσο αφορά τα προσωπικά στοιχεία του κάθε χρήστη
της υπηρεσίας χωρίς να υπάρχει η πιθανότητα κάποιος άλλος να έχει πρόσβαση σε αυτά.

# User Priority: (5/5) 
Η *<Λειτουργική Απαίτηση 2>* είναι βασική για τους διάφορους φορείς απασχόλησης αλλά και αυτούς που αναζητούν εργασία, 
καθώς χωρίς τη χρήση στοιχείων εισόδου δεν δεν θα υπάρχει η οργάνωση και η ασφάλεια που απαιτείται.

# Technical Priority: (5/5)
Η υλοποίηση της *<Λειτουργική Απαίτηση 2>* είναι αναγκαία διότι αποτελεί κύρια αρχή λειτουργίας της υπηρεσίας.

# Stability: (5/5)
Η εισαγωγή των στοιχείων και ο έλεγχος τους γίνεται με  συγκεκριμένο τρόπο που δεν είναι δυνατό να τροποποιηθεί.

[login_account.feature](https://github.com/nstamatak/1o-Paradoteo/blob/master/requirements/login_account.feature)


## *<Λειτουργική Απαίτηση 3>*

**Η υπηρεσία πρέπει να έχει μενού με κατηγορίες .**

# Περιγραφή: 
Το σύστημα πρέπει να είναι εύκολο στη χρήση του έχοντας απλοποιημένο μενού ,Έτσι  μειώνεται και ο χρόνος εξυπηρέτησης κάθε χρήστη .
Αυτό συντελεί στην αποδοτικότερη εξυπηρέτηση των χρηστών και μείωση του φόρτου των εξυπηρετητών που παρέχουν την υπηρεσία στο διαδίκτυο.

# User Priority: (5/5)
 Η *<Λειτουργική Απαίτηση 3>* είναι σημαντική για τους διάφορους φορείς απασχόλησης αλλά και αυτούς που αναζητούν εργασία, 
καθώς μια υπηρεσία εύκολη και γρήγορη στην χρήση είναι πιο επιθυμητή .

# Technical Priority: (5/5) 
Η υλοποίηση της *<Λειτουργική Απαίτηση 3>* θεωρείται μεγάλης σημασίας διότι όσο καλό και τελειοποιημένο είναι το περιεχόμενο που παρέχεται απο μια υπηρεσία ,δεν θα προτιμηθεί εαν είναι δύσχρηστη .

# Stability: (5/5) 
Ο αριθμός των επιλογών του μενού της υπηρεσίας μπορεί να μεταβάλλεται αναλόγως με τις απαιτήσεις κάθε περιόδου από τους χρήστες και των δυνατοτήτων που θέλουν οι διαχειριστές να παρέχουν .

## *<Λειτουργική Απαίτηση 5>*

**Οι διαχειριστές της υπηρεσίας πρέπει να μπορούν να επεξεργαστούν ή να διαγράψουν ορισμένες από τις
θέσεις εργασίας που έχουν δημοσιευθεί ήδη.**

# Περιγραφή: 
Οι διαχειριστές της υπηρεσίας πρέπει να μπορούν να επεξεργάζονται κάτι που έχει δημοσιευθεί με λανθασμένο τρόπο ή ακόμα και να το διαγράφουν ώστε να εμφανίζεται με σωστό τρόπο στους ενδιαφερόμενους, αλλά και να μην εμφανίζεται καθόλου όταν είναι αναγκαίο.

# User Priority: (4/5) 
Είναι σημαντικό οι διαχειριστές να μπορούν να κάνουν αλλαγές ή και να διαγράφουν τις θέσεις εργασίας που έχουν δημοσιευθεί, ώστε να μην υπάρχει λανθασμένη πληροφόρηση των ενδιαφερομένων.

# Technical Priority: (4/5) 
Είναι σημαντικό να υλοποιηθεί η συγκεκριμένη λειτουργία έτσι ώστε οι πληροφορίες οι οποίες παρέχονται να είναι αληθείς και ακριβείς.

# Stability: (5/5) 
Η διαδικασία της επεξεργασίας και της διαγραφής δεδομένων, παρά το γεγονός ότι επιρρεάζεται από την τεχνολογία η οποία χρησιμοποιείτε, παραμένει αμετάβλητη. Κατα συνέπεια δεν πρόκειται να μεταβληθεί ακούσια.

# Technical Priority: (5/5) 
Η υλοποίηση της *<Λειτουργική Απαίτηση 6>* θεωρείται αρκετά χρήσιμη διότι θα παρέχεται έτσι μια πρόσθετη και αρκετά βοηθητική πληροφορία στους ενδιαφερομένους .

# Stability: (5/5) 
Η δημοσίευση πραγματοποιείται με συγκεκριμένο τρόπο όπου δεν συνίσταται καμία μεταβολή αν και επηρεάζεται από την ανάπτυξη τεχνολογιών.

Η υπηρεσία προσφέρει στο χρήστη άμεση δήλωση ενδιαφέροντος προς κάποια θέση εργασίας μέσω του Διαδικτύου, στέλνοντας ηλεκτρονικά τα εγγραφα το οιποίο είναι απαραίτητη προϋπόθεση.

# Technical priority (5/5):
Η ορθή λειτουργία του συστήματος είναι εξαρτημένη με τη συμβατότητά της με τους περιηγητές ιστού.

# Stability (3/5):
Η απαίτηση αυτή είναι πολύ σταθερή και θα αλλάξει μόνο σε περίπτωση που οι τεχνολογίες που υποστηρίζουν οι browsers μεταβληθούν σημαντικά. 




## *<Μη λειτουργική Απαίτηση 8>*
**Το σύστημα πρέπει να διασφαλίζει ότι τα αρχεία των εγγεγραμμένων χρηστών δεν θα είναι προσβάσιμα από τρίτους, χωρίς την άδεια τους.**

# Περιγραφή:
Οι δημιουργοί του συστήματος πρέπει να παρέχουν τις πιο πρόσφατες τεχνολογίες ασφαλείας, ώστε να μην επιρρεστεί η εμπιστοσύνη των χρηστών.

# User priority (5/5):
Οι χρήστες ανεβάζουν αρχεία με ευαίσθητα προσωπικά δεδομένα. Με την  απαίτηση αυτή, θα υπάρχει αίσθηση ασφάλειας για την προσθήκη τέτοιου είδους αρχείων στο λογαριασμό του.

# Technical priority (5/5):
Υπάρχει αλληλένδετη σχέση της ασφάλειας των αρχείων του χρήστη και του συστήματος, πράγμα που την κάνει απολύτως αναγκαία για τους διαχειριστές.

# Stability (3/5):
Η *<Μη λειτουργική Απαίτηση 8>* σχετίζεται με τις τάσεις που επικρατούν στο διαδίκτυο σε σχέση με τις τεχνολογίες ασφάλειας.




## *<Μη λειτουργική Απαίτηση 9>*
**Ο εξυπηρετητής πρέπει να είναι σε συνεχή λειτουργία τουλάχιστο το 99% του χρόνου.**


# Περιγραφή: 
Ο εξυπηρετητής θα πρέπει να είναι διαθέσιμος στον χρήστη 99 στις 100 φορές.

# User Priority (5/5): 
Η απαίτηση αυτή θεωρείται αναγκαία για τον χρήστη, καθώς η συνεχής διαθεσιμότητα του συστήματος βελτιώνει την εμπειρία χρήσης.

# Technical Priority(5/5): 
Όσες αλλαγές επιθυμεί να κάνει η ομάδα των διαχειριστών στο σύστημα , αυτές γίνονται ορατές στο χρήστη, μόνο εφόσον ο εξυπηρετητής λειτουργεί.

# Stability(3/5): 
Σταθερή απαίτηση, που επηρεάζεται απότην ποιότητα των δικτύων και του υλικού εξοπλισμού.




## *<Μη λειτουργική Απαίτηση 10>*
**Οι διαχειριστές πρέπει να ταυτοποίουνται εκτός από το όνομα χρήστη και τον κωδικό πρόσβασης τους, με αναγνώριση δακτυλικού αποτιπόματος.**


# Περιγραφή: 
Λόγο των ευαίσθητων προσωπικών δεδομένων που θα είναι αποθήκευμένα μέσα στην υπηρεσία, θα χρησιμοποιηθεί ειδική συσκευή ταυτοποίησης δακτυλικού αποτυπώματος, για την ταυτοποίηση της ομάδας των διαχειριστών της υπηρεσίας με τον τρόπο αυτό διασφαλίζεται ότι κανένας τρίτος δεν θα εισέλθει έχοντας υποκλέψει το όνομα χρήστη (Username) και τον κωδικό πρόσβασης (password) ενός διαχειριστή και να χρησιμοποιήσει ως διαχειριστής την υπηρεσία.

# User Priority (5/5):
Στην υπηρεσία ανεβαίνουν πολλά ευαίσθητα προσωπικά δεδομένα. Συνέπεια αυτού είναι ότι πρέπει να τηρούνται τα ύψηστα μέτρα ασφαλείας προκειμένου να παρέχεται η μέγιστη αίσθηση ασφάλειας προς τους χρήστες.

# Technical Priority (3/5):
Το σύστημα θα λειτουργούσε κανονικά και με πολύ ασφάλεια ακόμα και χωρίς αύτην την ΜΛΑ. Ωστόσο εξαιτίας της σχέσης ασφάλειας των αρχείων του χρήστη και του συστήματος, κάνει ιδιαίτερα σημαντική την προσθήκη αυτή.

# Stability (3/5): 
Υπάρχουν περιθώρια μεταβολής στη *<Λειτουργική Απαίτηση 10>*, καθώς μπορεί να χρησιμοποιηθέι κάποιος άλλος επιπρόσθετος τρόπος ταυτοποίησης αντί του δακτυλικού αποτυπώματος ή ακόμα να βελτιωθεί και ο εξοπλισμός ο οποίος χρησιμοποιείται για να την υλοποιήσει κατά την πάροδο των ετών.
